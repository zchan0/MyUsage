# MyUsage site

Static site for the MyUsage project, built with [Astro](https://astro.build).
Hosted on GitHub Pages at https://zchan0.github.io/MyUsage/.

## Local development

```bash
cd site
npm install
npm run dev    # localhost:4321/MyUsage
```

`npm run build` runs the Astro build then `scripts/build-llms-full.mjs`,
which concatenates the docs + blog content into `dist/llms-full.txt` for
LLM crawlers.

## Deployment

GitHub Actions (`.github/workflows/deploy-site.yml`) builds the site on
every push to `main` and deploys via the `actions/deploy-pages@v4`
action. Requires the repository's GitHub Pages source to be set to
**"GitHub Actions"** in *Settings → Pages* (one-time setup).

## Structure

```
site/
├── public/                 # Static assets served verbatim
│   ├── robots.txt          # Segmented by AI crawler type (per Tw93's GEO playbook)
│   └── llms.txt            # Site overview for AI crawlers (llms.txt standard)
├── src/
│   ├── layouts/            # Shared HTML shells
│   ├── pages/              # Each .astro file = one route
│   ├── components/         # Reusable UI (e.g. Hero, Card)
│   ├── content/            # Markdown content collections (docs + blog)
│   └── styles/             # Global CSS
├── scripts/
│   └── build-llms-full.mjs # Combines docs + blog into a single MD for LLMs
└── astro.config.mjs
```

## SEO / GEO conventions

- **`robots.txt`** segments AI crawlers: allow search/retrieval +
  user-triggered, block training + undeclared. (See [Tw93's article][tw93].)
- **`llms.txt`** at site root, Markdown overview for LLM crawlers per
  the [llms.txt standard](https://llmstxt.org).
- **`llms-full.txt`** at site root (auto-built), 30–60 KB single-file
  bundle of docs + blog content.
- **`<link rel="alternate" type="text/markdown">`** on every page
  pointing at the page's `.md` source so Claude Code / Cursor /
  retrieval crawlers can grab clean Markdown.
- **Sitemap** auto-generated by `@astrojs/sitemap`, submitted to Google
  Search Console and Bing Webmaster Tools after deploy.
